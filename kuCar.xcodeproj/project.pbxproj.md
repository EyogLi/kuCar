# Xcode Project Setup Guide

This file provides instructions for setting up the Xcode project on a Mac.  
The project.pbxproj file must be generated by Xcode.

## Prerequisites

- macOS 14+ (Sonoma or later)
- Xcode 15.4+  
- iOS 17.0+ SDK

## Setup Steps

### 1. Create a New Xcode Project

1. Open Xcode → File → New → Project
2. Select **iOS → App**
3. Configure:
   - Product Name: `kuCar`
   - Team: Your Apple Developer Team
   - Organization Identifier: (your reverse domain, e.g., `com.yourcompany`)
   - Interface: **SwiftUI**
   - Language: **Swift**
   - Minimum Deployment: **iOS 17.0**
   - Storage: **SwiftData** (check the box)
4. Save the project at `d:/kuCar/` (or copy files later)

### 2. Add Source Files to Project

After creating the project, add all source directories to the Xcode project:
- Drag the following folders into Xcode (Create groups, Add to target):
  - `kuCar/App/`
  - `kuCar/Core/`
  - `kuCar/Features/`

### 3. Add Resources

- Drag `kuCar/Resources/` into Xcode (Create folder references)
- Include `.metal` files in Compile Sources build phase
- Include `.mlpackage` files in Copy Bundle Resources
- Include `car_database.json` in Copy Bundle Resources

### 4. Configure Build Settings

| Setting | Value |
|---------|-------|
| iOS Deployment Target | 17.0 |
| Swift Language Version | Swift 5.9 |
| Enable Bitcode | NO |
| Metal Compiler - Build Options | Standard |
| Metal Framework Search Paths | `$(DEVELOPER_DIR)/Platforms/iPhoneOS.platform/Developer/SDKs/iPhoneOS.sdk/System/Library/Frameworks` |

### 5. Add Framework Dependencies

In Target → General → Frameworks, Libraries, and Embedded Content:
- `CoreML.framework` (Do Not Embed)
- `Vision.framework` (Do Not Embed)
- `CoreImage.framework` (Do Not Embed)
- `Metal.framework` (Do Not Embed)
- `MetalKit.framework` (Do Not Embed)
- `SwiftData.framework` (Do Not Embed)
- `PhotosUI.framework` (Do Not Embed)
- `AVFoundation.framework` (Do Not Embed)
- `UniformTypeIdentifiers.framework` (Do Not Embed)

### 6. Download DeepLabV3 Model

1. Download the CoreML version of DeepLabV3 from:
   https://developer.apple.com/machine-learning/models/
2. Place `DeepLabV3.mlpackage` in `kuCar/Resources/CoreMLModels/`
3. Add it to Xcode target (it will be auto-compiled by CoreML)

### 7. Privacy Info

The project already includes `PrivacyInfo.xcprivacy` in Resources.  
Make sure it's included in Copy Bundle Resources.

### 8. App Icon

Add app icon assets to `Resources/Assets.xcassets/AppIcon.appiconset/`:
- 1024x1024 (App Store)
- All required sizes for iPhone

## DeepLabV3 Model Notes

The `CarSegmentationService.swift` file references `DeepLabV3` class.  
After adding the `.mlpackage` to Xcode, CoreML will auto-generate the Swift class.

Expected class name: `DeepLabV3`
Expected input: Image (513x513, CVPixelBuffer)
Expected output: semanticPredictions (MultiArray or CVPixelBuffer)

## WheelDetector Model (Phase 3)

For Phase 3, train a custom wheel detector using Create ML:
1. Open Create ML app
2. Create new Object Detection project
3. Import training images with bounding box annotations
4. Train and export as CoreML model
5. Add `WheelDetector.mlpackage` to the project

## Testing

1. Build the project in Xcode (⌘+B)
2. Run on simulator or device (⌘+R)
3. Test with car photos

## App Store Submission

When ready to submit:
1. Archive the project (Product → Archive)
2. Upload to App Store Connect
3. Fill in App Store listing (name, description, screenshots, keywords)
4. Include in App Review Notes:
   ```
   This app uses on-device CoreML models for car segmentation.
   All image processing is performed locally.
   No images are uploaded to any server.
   ```
